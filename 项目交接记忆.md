# whatlanCar 项目交接记忆

## 项目概况

这是一个 Windows WinForms 工具，用于在 VMware 内运行的 FPS 类游戏中做视觉辅助、YOLO 攻击测试、数据采集、策略模型训练与 AI 驾驶。

核心能力：
- 捕获 VMware 游戏窗口画面。
- 使用 Depth Anything ONNX 生成深度图。
- 使用 UNIFY/YOLO 做目标检测攻击。
- 通过开发板串口输出键盘控制。
- 通过双头鼠标/`msdk.dll` 做 FPS 视角相对转向。
- 采集图像、深度、小地图和动作数据，训练策略 ONNX 模型。

## 主要文件

- `MainWindow.cs`：主窗口、YOLO 攻击、AI 驾驶、模型选择、AI 预览、卡住脱困。
- `DataCollectorWindow.cs`：数据采集窗口、训练入口、预览面板、动作与帧清单保存。
- `PolicyInference.cs`：策略 ONNX 加载与推理，读取 metadata 判断是否需要深度/小地图。
- `training/train_policy.py`：PyTorch 训练脚本，导出 ONNX、PT、JSON metadata。
- `tools/vm_key_sender.ps1`：虚拟机内运行的键鼠状态 UDP 发送脚本。
- `DevBoardController.cs`：开发板串口键盘协议封装。
- `Esp32S3UsbKeyboardFirmware.ino`：ESP32-S3 USB 键盘固件。
- `UnifyControlBridge.cs`：UNIFY、YOLO、VMware、双头鼠标封装。
- `VmwareWindowHelper.cs`：VMware/MKS 窗口查找、移动、嵌入。
- `WindowCapture.cs`：窗口截图工具。

## 数据采集

默认数据目录：

```text
C:\Users\Administrator\Documents\whatlanCar-dataset
```

采集内容按勾选项保存：
- `frames/`：游戏截图。
- `depth/`：深度可视化图。
- `minimap/`：小地图裁剪图。
- `actions.csv`：按键和鼠标状态。
- `frame_manifest.csv`：每帧图像路径与动作标签。

采集窗口行为：
- 不勾深度图时，不显示深度预览，也不跑深度推理预览。
- 不勾小地图时，不显示小地图预览，也不裁剪小地图预览。
- 小地图耗时使用小数毫秒显示，避免太快时看起来一直是 0ms。
- 已移除无必要的宿主机 RawInput 注册/注销逻辑，避免停止采集时报错误 87。

## 虚拟机键鼠发送

虚拟机内运行：

```powershell
powershell -ExecutionPolicy Bypass -File tools\vm_key_sender.ps1
```

脚本通过 UDP `47891` 向宿主机发送：
- `w/a/s/d/jump/m`
- `mouse_dx/mouse_dy`
- `mouse_x/mouse_y`
- `mouse_source`

脚本优先使用 Raw Mouse Input 获取相对鼠标位移，解决 FPS 锁鼠标时光标坐标不动、`mouse_dx` 始终为 0 的问题。

## 策略训练

训练脚本：

```text
training/train_policy.py
```

训练前会打印：
- `move(stop,forward,back)`
- `turn(none,left,right)`
- `jump(no,yes)`
- `mouse_dx nonzero/min/max/mean_abs`

如果没有任何左右转向标签，训练会直接报错，防止生成只会按 W 的坏模型。

当前支持三类模型：
- `policy_rgb.onnx`：纯画面。
- `policy_rgb_depth.onnx`：画面 + 深度。
- `policy_rgb_depth_minimap.onnx`：画面 + 深度 + 小地图。

采集窗口会按勾选项自动输出对应模型。不勾深度时会自动禁用小地图输入，避免生成“画面+小地图但无深度”的第四种模型。

## AI 驾驶

主窗口新增模型下拉框：
- 纯画面。
- 画面+深度。
- 画面+深度+小地图。

AI 驾驶逻辑：
- 策略模型输出 move/turn/jump。
- 执行前经过规则层过滤。
- 模型不需要深度/小地图时，AI 循环不会额外跑深度推理或小地图裁剪，降低延迟。
- 有左/右转向动作时，先松开 `W/S`，只做鼠标转向；下一轮再决定是否继续前进。
- 前进采用脉冲式 W，降低过快移动造成的延迟不匹配。
- 卡住时自动后退，再原地大角度转向脱困。
- AI 预览显示当前画面、动作、置信度、turn margin、RECOVERY 状态和方向箭头，约 200ms 刷新一次。

## 开发板与固件

`DevBoardController.ReleaseAll()` 已增强：
- 先显式发送 `_w/_a/_s/_d/_j/_m`。
- 再发送 `!`。

`Esp32S3UsbKeyboardFirmware.ino` 已增加：

```cpp
else if (c == '!') {
  Keyboard.releaseAll();
}
```

注意：需要重新刷开发板固件，硬件端 `!` 才会真正生效。即使未刷固件，PC 端显式 `_w` 等释放命令也能提升停止 AI 后释放 W 的可靠性。

## 录制经验

不要把走路和跑步数据直接混在同一个模型里。相似画面下二者需要不同动作节奏，混训容易学成不稳定的中间态。

建议：
- 先录步行模型，把门口、拐角、卡住恢复跑通。
- 跑步模型单独录制、单独训练。
- 训练和执行速度保持一致。

样本量经验：
- 600 帧只适合验证数据链路。
- 2000-4000 帧适合快速验证。
- 8000-15000 帧适合较稳定的短路线模型。
- 多地图或泛化需求，需要按地图/速度/输入类型分开录制更多数据。
